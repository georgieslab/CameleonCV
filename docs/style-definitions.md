## 1. Professional

### Core Intent
Communicate competence and reliability through industry-standard language. Project the image of someone who understands corporate norms and can be trusted in client-facing situations.

### Linguistic Markers

| Marker | Example |
|--------|---------|
| Formal register | No contractions, measured tone |
| Action verbs | "Led," "Developed," "Implemented," "Coordinated" |
| Quantified impact | "Reduced costs by 15%," "Managed team of 8" |
| Third-person orientation | Focus on role/outcomes, not personal narrative |
| Parallel structure | Consistent grammatical patterns across bullets |

### Anti-Patterns

- âŒ Casual phrases: "stuff," "things," "a lot of"
- âŒ Emotional language: "passionate about," "love working with"
- âŒ Vague claims: "improved efficiency" (without metrics)
- âŒ First-person narrative: "I really enjoyed..."
- âŒ Exclamation marks

### Boundary Conditions

**Crosses into "stiff"** when:
- Passive voice dominates ("Results were achieved...")
- Jargon obscures rather than clarifies
- All personality is stripped away
- Reads like a legal document

### Target Audience

- Corporate hiring managers
- Traditional industries (finance, consulting, law)
- Formal HR screening processes
- Large enterprise environments

### Example Transformation

**Before:**
> I did a lot of data analysis stuff and made some reports.

**After:**
> Conducted data analysis and developed business intelligence reports for senior stakeholders.

## 2. Academic

### Core Intent
Demonstrate intellectual rigor and scholarly credibility. Signal that you operate within academic norms and can contribute to research-oriented environments.

### Linguistic Markers

| Marker | Example |
|--------|---------|
| Methodological precision | "Employed mixed-methods approach," "Utilized regression analysis" |
| Hedged claims | "Contributed to," "Evidence suggests," "Findings indicate" |
| Nominalization | "The implementation of" rather than "implemented" |
| Complex sentence structures | Subordinate clauses, academic transitions |
| Domain-specific terminology | Field-appropriate technical vocabulary |

### Anti-Patterns

- âŒ Promotional language: "cutting-edge," "revolutionary," "world-class"
- âŒ Unsubstantiated superlatives: "best," "leading," "top"
- âŒ Oversimplification of complex processes
- âŒ Corporate buzzwords: "synergy," "leverage," "disrupt"
- âŒ Exclamation marks or casual emphasis

### Boundary Conditions

**Crosses into "inaccessible"** when:
- Jargon prevents comprehension by educated non-specialists
- Sentence complexity impedes readability
- Hedging undermines all confidence
- Reads like a dissertation abstract

### Target Audience

- Academic hiring committees
- Research institutions
- Grant review panels
- Scholarly publication contexts

### Example Transformation

**Before:**
> I did a lot of data analysis stuff and made some reports.

**After:**
> Conducted systematic quantitative analyses employing statistical methodologies, synthesizing findings into comprehensive research documentation.

## 3. Confident

### Core Intent
Project self-assurance and leadership presence. Signal that you take ownership, deliver results, and operate with executive-level conviction.

### Linguistic Markers

| Marker | Example |
|--------|---------|
| Direct assertions | "Delivered," "Achieved," "Drove" â€” no hedging |
| Ownership language | "I built," "I led," "Under my direction" |
| Scale emphasis | Team sizes, budget figures, geographic reach |
| Outcome-first structure | Lead with results, then explain how |
| Short, declarative sentences | Punchy rhythm, no meandering |

### Anti-Patterns

- âŒ Hedging language: "helped to," "assisted with," "contributed to"
- âŒ Passive voice: "The project was completed"
- âŒ Minimizing qualifiers: "just," "only," "a small team"
- âŒ Permission-seeking tone: "I was given the opportunity to..."
- âŒ Burying achievements in long sentences

### Boundary Conditions

**Crosses into "arrogant"** when:
- Claims cannot be substantiated
- Team credit is individually claimed
- Every sentence starts with "I"
- Reads like bragging rather than reporting

### Critical Rule: No Fabrication

Confident style transforms **how** achievements are presented, not **what** is claimed. 

| Original | Allowed | NOT Allowed |
|----------|---------|-------------|
| "Helped with the project" | "Executed project deliverables" | "Led the project" |
| "Part of a team" | "Team member delivering..." | "Team lead" |
| "Around 30%" | "30%" (remove hedging) | "35%" (changing numbers) |

### Target Audience

- Executive roles
- Leadership positions
- Competitive industries
- Direct-to-hiring-manager applications

### Example Transformation

**Before:**
> I did a lot of data analysis stuff and made some reports.

**After:**
> Delivered data analysis and actionable reports that informed key business decisions.

## 4. Concise

### Core Intent
Maximize information density while minimizing word count. Respect the reader's time by making every word earn its place.

### Linguistic Markers

| Single-clause sentences | Avoid compound/complex structures |
| Verb-first bullets | "Built," "Shipped," "Cut costs by 30%" |

### Anti-Patterns

- âŒ Redundant modifiers: "completely finished," "very unique"

### Boundary Conditions

**Crosses into "cryptic"** when:

- Acronyms aren't explained
- Reads like bullet fragments

### Target Audience

- Technical recruiters scanning quickly
- High-volume hiring processes
- Roles where brevity signals competence
- Supplementary to longer documents

### Example Transformation

**Before:**

## Validation Criteria

When evaluating style transformations, check:

### Style Fidelity (Score 1-5)
- Are the linguistic markers present?
- Are anti-patterns avoided?
- Is the style consistent throughout?
- Would a reader immediately recognize the target style?

### Factual Consistency (Score 1-5)
- Are all facts from the original preserved?
- Are metrics identical (no rounding, no additions)?
- Are role descriptions accurate (no inflation)?
- Are no new claims introduced?

### Quality (Score 1-5)
- Is the output grammatically correct?
- Is it suitable for the target audience?
- Would you submit this in a real application?
- Does it meet ATS requirements (where applicable)?

## Usage Notes

### Combining Styles
Users may want different styles for different CV sections:
- Summary: Confident
- Experience: Professional
- Skills: Concise
- Projects: Playful

This is valid â€” the model should handle section-level style selection.

### Style Selection Guidance

| If applying to... | Consider... |
|-------------------|-------------|
| Big 4 consulting | Professional |
| University research position | Academic |
| C-suite role | Confident |
| Technical screening | Concise |
| Early-stage startup | Playful |

### When Transformation Is Minimal

Sometimes the original text already matches the target style. In these cases:
- Minor refinements are acceptable
- Zero-change outputs are valid training data (teaches restraint)
- Flag as "low transformation" in metadata
